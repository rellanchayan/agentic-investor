## Lessons

1. A pattern that has now repeated four times in six days is a structural problem, not a
   discipline lapse. The morning decision file is missing every day except the launch day and
   the Juneteenth holiday record. Writing the rule down has not fixed it. The fix must be
   mechanical: the morning phase script should write an empty decision file as a stub at startup
   and exit non-zero if it remains unpopulated by close. Logging a rule in the PLAYBOOK is
   necessary but not sufficient.

2. SPY has now fallen 2.26% since we launched. Our paper equity is flat. On paper that looks
   like we are winning. We are not winning — we are not playing. Every day that passes without
   a morning decision record is a day the process is not running at all. The account cannot
   learn, improve, or beat SPY over years if the core daily loop does not execute. The fill
   rate, thesis quality, and sell discipline are all secondary to simply running the process.

3. `state/equity_history.jsonl` does not exist as of this writing. The equity snapshot step
   either has not run or has written to a different path. Without this file, trend data (e.g.,
   equity over time, drawdown history) cannot be reconstructed. The EOD script should verify
   that the snapshot is written and surface an error if it is missing.
