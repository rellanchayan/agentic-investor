## Lessons

1. Four consecutive trading days without a morning decision file (2026-06-22 through
   2026-06-25) is no longer a pattern to note — it is a process breakdown. The PLAYBOOK has
   stated since 2026-06-18 that a decision file is mandatory every trading day. Restating
   the rule a third time in this journal will not fix anything. The fix must be structural:
   the morning phase script should refuse to exit cleanly unless `docs/decisions/<date>.md`
   exists. Until that enforcement is in place, the journal can only document the absence.

2. Two full cycle gaps (2026-06-23 and 2026-06-24 have neither a morning decision file nor
   an EOD summary) mean two trading days where the system produced zero output of any kind.
   This is a different failure mode from a missing decision file: it means the entire daily
   cycle did not run. A journal entry cannot reconstruct what should have happened. It can
   only record that the gap exists and that the numbers from those days are unrecoverable
   from this system's own records.

3. Cash held while the market falls looks safe. It is not an achievement. SPY is down 2.30%
   since we opened the account on 2026-06-16. Our 0.00% total return is not a result of
   good stock selection or patient discipline — it is the result of the system not buying
   anything. When SPY recovers, this gap will vanish. Real performance requires owning
   something that compounds. Nine days in, the account has never owned a single share.
